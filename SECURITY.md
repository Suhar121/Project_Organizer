# Security Policy

## Reporting a Vulnerability

Please do not open public issues for security vulnerabilities.

Instead, report privately with:

- A clear description of the issue
- Reproduction steps or proof of concept
- Potential impact
- Suggested remediation (optional)

If GitHub Security Advisories are enabled for this repository, use that flow.

## Response Expectations

- Initial triage response target: within 5 business days
- Remediation timeline depends on severity and complexity

## Scope Notes

This project is local-first and intended for developer machines. Security issues are still important, especially around command execution and secret storage.
